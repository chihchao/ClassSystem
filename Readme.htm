ClassSystem V2 Readme
（for PHP4.2）
系統需求 修改事項 升級方式 安裝方式 版本修改
  • 系統需求
  1. PHP4
  2. MySQL
  • 修改事項
  1. 修正在 PHP4.2 以後版本，讀取不到變數及出現警告字句的問題。
  2. 修正在 PHP4.2 以後版本，編輯文件、相片、管理設定時，如未上傳檔案，會誤刪附檔及圖片的問題。
  3. 解決 UTF-8 中文檔案名稱無法連結的問題。
  4. 新增教師可自行刪除首頁圖片的功能。
  5. 新增首頁圖片及相片區可採用 Flash 的 SWF 檔。
  6. 新增刪除文件附檔的功能。
  7. 新增搬移連結的功能。
  8. 新增編輯行事曆各項記事的功能。
  9. 可直接在 PublicParameter.php 中修改年級及班級名稱。
  • 升級方式
  1. 進入 MySQL 的班級網頁資料庫，執行 ClassSystemV2_update.sql。
  2. 刪除原有目錄中的 PHP 檔及 Images 目錄（不要刪除 UploadDocument, UploadHomepage, UploadPhoto 內的檔案），然後將 ClassSystemV2.zip 解壓縮，並將目錄下的 PHP 檔及 Images 目錄複製到原本安裝的目錄中。
  3. 修改 PublicParameter.php
    1. 修改連結資料庫的使用者名稱，密碼，資料庫名稱($DATABASE_URL; $DATABASE_USER; $DATABASE_PASSWORD; $DATABASE_BASE)。
    2. 修改管理所有網頁密碼($MANAGE_PASSWORD; 管理者可刪除某班班級頁或更改班級教師密碼)。
    3. 修改申請班級網頁密碼($APPLY_PASSWORD)。
    4. 修改年級名稱及班級名稱($GRADE_ARRAY; $CLASS_NUMBER_ARRAY)。
  • 安裝方式
  1. 將程式解壓縮放在 web 伺服器網頁目錄下。
  2. 在 MySQL 中建一資料庫。
  3. 在 MySQL 剛建立的資料庫中執行 ClassSystemV2.sql 建立資料表格。
  4. 修改 PublicParameter.php
    1. 修改連結資料庫的使用者名稱，密碼，資料庫名稱($DATABASE_URL; $DATABASE_USER; $DATABASE_PASSWORD; $DATABASE_BASE)。
    2. 修改管理所有網頁密碼($MANAGE_PASSWORD; 管理者可刪除某班班級頁或更改班級教師密碼)。
    3. 修改申請班級網頁密碼($APPLY_PASSWORD)。
    4. 修改年級名稱及班級名稱($GRADE_ARRAY; $CLASS_NUMBER_ARRAY)。
  5. 修改 UploadDocument 的權限，目錄擁有者改為 apache。
  6. 修改 UploadHomepage 的權限，目錄擁有者改為 apache。
  7. 修改 UploadPhoto 的權限，目錄擁有者改為 apache。
  8. 上傳檔案的大小受限於 php.ini 的 upload_max_filesize，內定是 2M，如果想讓老師上傳更大的檔案，就更改這個設定值並重新啟動 apache 即可。
  • 版本修改v2.1 -- By 蔡俊彥
    1. 修改HTTP HEADER認證方式，改以Web界面認證，使IIS+PHP4系統可以使用。
    2. 選單增加登出功能。
    3. 管理區增加可以選擇開放功能。
